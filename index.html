Technical Notes & Learnings

Real-Time Systems

  • Spark Streaming vs Flink: Spark works well for micro-batching, but Flink is non-negotiable for true low-latency, stateful processing.
  • Handling Kafka consumer backpressure during traffic spikes without lag or message loss.

Production Engineering

  • Batch vs streaming trade-offs: Don’t stream unless latency is a hard requirement—the complexity cost is high.
  • Idempotency is key in distributed pipelines to handle retries without data duplication.

Data Quality

  • Implementing circuit breakers in Airflow to stop bad data from polluting downstream tables.
  • Shift-left testing: Catching schema changes before they break the warehouse.
